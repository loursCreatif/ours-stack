# Presets — shared with layout-html

Do **not** duplicate full CSS. Load palette tokens from [`layout-html/references/preset-palettes.md`](../../layout-html/references/preset-palettes.md) and shell notes from [`layout-html/references/presets/`](../../layout-html/references/presets/).

## Mapping for mind-map nodes

| Node type | CSS var |
|-----------|---------|
| `concept` | `--accent` fill, white text |
| `detail` | `--surface` fill, `--text` color, `--accent` border |
| `example` | `--success` border, `--accent-light` fill |
| `source` | `--secondary` fill, underline on hover |

## Preset → `:root` tokens

Inject the matching block from `html-template.md` § Preset CSS when composing HTML.

| Preset | Reference HTML |
|--------|----------------|
| `warm` | `report-warm.html` |
| `editorial` | `report-editorial.html` |
| `minimal` | `report-minimal.html` |
| `creative` | `report-creative.html` |

Legacy `academic` maps may still be regenerated by the composer, but do not offer `academic` in the style question for new maps.

## Custom preset

User describes style in chat → map to `:root` vars per `layout-html/references/style-presets.md` § Custom. Set `meta.preset: "custom"` **and** `meta.customTokens` (see `node-schema.md` § Custom preset — token names, safe values). `compose-html.py` injects them over `warm` defaults; without `customTokens` the map silently renders `warm`, so always write them.
